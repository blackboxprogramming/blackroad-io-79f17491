Quick Start

5 min read
Updated Jan 15, 2024

This guide will help you get up and running with BlackRoad in under five minutes. You'll create your first agent and understand the basic concepts of the system.

Prerequisites
Before starting, ensure you have Node.js 18+ and npm installed on your system.

Installation

Install the BlackRoad CLI globally using npm. This provides you with the blackroad command for managing your projects.

# Install BlackRoad CLI
npm install -g @blackroad/cli

# Initialize a new project
blackroad init my-project

# Start the development server
cd my-project && blackroad dev

Creating Your First Agent

Agents are the core building blocks of BlackRoad. Each agent has its own memory, logic engine, and can communicate with other agents in the network.

import { Agent } from '@blackroad/core';

const agent = new Agent({
  name: 'Lucidia',
  role: 'Core Intelligence',
  memory: {
    type: 'journal',
    persistence: true
  }
});

await agent.start();

Agent Configuration

The agent constructor accepts several configuration options. Here's a complete reference:

Option Type Default Description
name string — Unique identifier for the agent
role string — Description of the agent's function
memory object {} Memory configuration object
logic string "trinary" Logic engine type (trinary, binary)

Memory System

BlackRoad uses an append-only journal system for memory persistence. This ensures complete auditability and prevents data corruption through immutable logging.

Tip
For development, you can use mutable memory mode for faster iteration. Switch to journal mode for production deployments.
  • Append-only journal — All memory operations are logged immutably
  • PS-SHA hashing — Cryptographic verification of memory integrity
  • Automatic persistence — Memory state is saved on every operation
  • Point-in-time recovery — Restore to any previous state
