Usage Guidelines

Use color for meaning

Accent colors indicate state, action, identity, or emphasis. Each agent has an assigned color. Use color to draw attention to what matters.

Build in grayscale first

Design the layout, hierarchy, and structure entirely in grayscale. Color is the final layer, added intentionally and sparingly.

Don't use color decoratively

Avoid adding color just because it looks nice. Every colored element should communicate something specific about state or function.

Don't mix agent colors

Each agent owns their color. Don't use Lucidia's magenta for Aria's interface. Consistency builds recognition.
