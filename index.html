Core Concepts

Understand the fundamental building blocks of BlackRoad OS: the three-layer architecture, agent coordination, and memory persistence systems that power recursive intelligence.

Last updated: Jan 2025 ~8 min read

Three-Layer Architecture

BlackRoad OS is built on a three-layer architecture that separates concerns between fundamental physics, canonical world state, and the buildable platform layer. This separation allows agents to operate with consistent rules while developers build custom experiences on top.

Key Insight

The three layers mirror the relationship between physics, reality, and human-built systems. Each layer has its own rules, but depends on the layers beneath it.

Universe Layer

The Universe layer defines the fundamental physics of the system. It establishes causality, information flow, and existence rules that all other layers must respect. This layer is immutable once deployed.

typescript
1import { Universe } from '@blackroad/core' 2 3// Universe defines fundamental constraints 4const universe = Universe.create({ 5 causality: 'strict', 6 information: 'conserved', 7 time: 'forward-only' 8})

Lucidia Core

The Lucidia layer is the canonical world where agents live. It implements the rules defined by Universe and provides the identity, memory, and coordination systems that agents use to interact with each other and their environment.

Metaverse Platform

The Metaverse layer is the buildable platform where developers create experiences. It provides the runtime and SDK for building applications that leverage Lucidia agents and Universe physics.

Agent System

Agents are the primary actors in BlackRoad OS. Each agent has a unique identity, persistent memory, and operates using trinary logic that extends beyond traditional boolean states.

Logic State Value Meaning
TRUE 1 Confirmed knowledge
FALSE 0 Confirmed negation
UNKNOWN -1 Uncertain or contradictory
Important

The UNKNOWN state is not a lack of information—it's an explicit acknowledgment of uncertainty that the system can reason about.

Memory Persistence

BlackRoad uses the PS-SHA∞ hashing system for memory persistence. All memory operations are append-only, creating an immutable journal of truth state commits.

typescript
1import { Memory } from '@blackroad/sdk' 2 3// Configure persistent memory 4const memory = Memory.persistent({ 5 hash: 'ps-sha-infinity', 6 journal: 'append-only', 7 commits: 'truth-state' 8}) 9 10// Memory commits are immutable 11await memory.commit({ 12 key: 'agent.state', 13 value: currentState, 14 timestamp: Date.now() 15})

API Reference

The core API provides methods for creating agents, managing memory, and coordinating between nodes in the cluster.

POST /agents/create

Create a new agent with the specified configuration.

name string required
Unique identifier for the agent. Must be lowercase alphanumeric with hyphens.
logic string optional
Logic system to use. Options: binary, trinary. Default: trinary
memory object optional
Memory configuration object. See Memory Persistence section for details.
